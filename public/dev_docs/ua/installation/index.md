# Керівництво з встановлення — Огляд

Це керівництво проведе вас через налаштування власної мережі SAVVA з нуля. Виконуючи ці інструкції, ви зможете запустити повноцінний екземпляр SAVVA з бекенд-сервером та фронтенд-інтерфейсом.

## Що ви розгорнете

Повноцінна мережа SAVVA складається з:

1. **Backend Server** - API-сервер на Go, який обробляє:
   - Аутентифікацію користувачів та сесії
   - Збереження та отримання постів
   - Інтеграцію з IPFS
   - Управління базою даних
   - WebSocket-з'єднання
   - Взаємодію з блокчейном

2. **UI Website** - фронтенд на SolidJS, який забезпечує:
   - Інтерфейс користувача для створення та перегляду контенту
   - Інтеграцію Web3-гаманців
   - Завантаження файлів у IPFS
   - Взаємодію зі смарт-контрактами
   - Підтримку кількох мов
   - SEO-видимість: серверно-генерований HTML для пошукових систем (Google, Bing, Yandex, Baidu, DuckDuckGo, Apple), AI-сканерів (GPTBot, ClaudeBot, PerplexityBot, Google-Extended, ...), та unfurl-інструментів (Telegram, X, Facebook, Discord, Slack, WhatsApp, ...), з файлом `robots.txt` і sitemap-ами для кожного домену

3. **Smart Contracts** :
   - Контракт токена SAVVA
   - Контракт стейкінгу
   - Контракт управління
   - Контракт NFT для контенту
   - Та інші...

## Огляд вимог

- **Server**: Сервер під управлінням Linux (рекомендується Ubuntu 20.04+)
- **Database**: PostgreSQL 14+ (або керована сервісом база даних)
- **IPFS**: Локальний вузол IPFS + зовнішній сервіс pinning з публічним gateway
- **Web Server**: Nginx або Apache
- **Domain**: Доменне ім'я з SSL-сертифікатом
- **Blockchain**: RPC-доступ до сумісної з Ethereum мережі (рекомендовано WSS)
