# Посібник з встановлення — огляд

Цей посібник проведе вас крок за кроком через налаштування власної мережі SAVVA з нуля. Виконуючи ці інструкції, ви зможете запустити повноцінний екземпляр SAVVA з серверною частиною (backend) та веб-інтерфейсом (frontend).

## Що ви розгортаєте

Повна мережа SAVVA складається з:

1. **Backend Server** — API-сервер на Go, що відповідає за:
   - автентифікацію користувачів і сесії
   - збереження та отримання постів
   - інтеграцію з IPFS
   - управління базою даних
   - WebSocket-з’єднання
   - взаємодію з блокчейном

2. **UI Website** — фронтенд на SolidJS, який забезпечує:
   - інтерфейс для створення та перегляду контенту
   - інтеграцію з Web3-гаманцями
   - завантаження файлів в IPFS
   - взаємодію зі смарт-контрактами
   - підтримку кількох мов

3. **Смарт-контракти**:
   - контракт токена SAVVA
   - контракт стекингу
   - контракт управління (governance)
   - контракт NFT для контенту
   - і інші...

## Огляд вимог

- **Сервер**: сервер під Linux (рекомендується Ubuntu 20.04+)
- **База даних**: PostgreSQL 14+ (або керований сервіс баз даних)
- **IPFS**: локальний вузол IPFS + зовнішній pinning-сервіс з публічним gateway
- **Веб-сервер**: Nginx або Apache
- **Домен**: доменне ім'я з SSL-сертифікатом
- **Блокчейн**: доступ до RPC мережі, сумісної з Ethereum (рекомендується WSS)
