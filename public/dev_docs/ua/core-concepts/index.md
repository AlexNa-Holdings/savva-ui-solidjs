# Основні концепції

Цей розділ охоплює основну архітектуру додатка SAVVA SolidJS: як він підключається до бекенду, як працюють домени та активи, як інтерфейс керується конфігурацією для кожного домену та інше.

## Теми

- [Ініціалізація додатка](/docs/core-concepts/app-init) - Як додаток завантажується і знаходить правильний бекенд та домен
- [Аутентифікація](/docs/core-concepts/auth) - Аутентифікація користувача та підключення гаманця
- [Контракт конфігурації](/docs/core-concepts/config-contract) - Як ` /info` керує поведінкою під час виконання
- [Оркестрація з'єднань](/docs/core-concepts/connection-orchestration) - Керування WebSocket і HTTP з'єднаннями
- [Домени](/docs/core-concepts/domain) - Конфігурація домену та розв'язання активів
- [Публікація дописів](/docs/core-concepts/publishing-posts) - Як дописи створюються та публікуються в IPFS
- [Відображення дописів](/docs/core-concepts/showing-posts) - Як дописи відображаються та рендеряться
- [Зашифровані дописи](/docs/core-concepts/encrypted-posts) - Наскрізне шифрування (end-to-end) для контенту, доступного лише підписникам
- [Профіль користувача](/docs/core-concepts/user-profile) - Управління профілем користувача та дані на блокчейні
