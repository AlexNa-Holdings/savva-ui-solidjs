что на это свободное программное обеспечение не распространяется гарантия.
Для блага пользователей и авторов GPL требует, чтобы изменённые версии были помечены
как изменённые, чтобы их проблемы не приписывались ошибочно авторам предыдущих версий.

изменённые версии программного обеспечения внутри них, хотя производитель может это делать.
Это принципиально несовместимо с целью защиты свободы пользователей изменять программное обеспечение.
Систематическая практика такого злоупотребления встречается в области продуктов для личного использования,

на эти области в будущих версиях GPL, чтобы защитить свободу пользователей.

  Наконец, каждую программу постоянно угрожают патенты на программное обеспечение.
Государства не должны разрешать патенты ограничивать разработку и использование

могут сделать её фактически проприетарной. Чтобы предотвратить это, GPL гарантирует,
что патенты не могут использоваться для превращения программы в несвободную.

  1. Исходный код.

  «Исходный код» для произведения означает предпочтительную форму произведения
для внесения изменений. «Объектный код» — любая форма произведения, не являющаяся исходным кодом.

  «Стандартный интерфейс» — интерфейс, который либо является официальным стандартом,
определённым признанным органом по стандартам, либо, в случае интерфейсов,
указанных для конкретного языка программирования, широко используется среди разработчиков
этого языка.

  «Системные библиотеки» исполняемого произведения включают всё, кроме самого произведения,
что (а) включено в обычную форму упаковки основного компонента, но не является частью этого компонента,
и (б) служит только для использования произведения с этим основным компонентом или для реализации
стандартного интерфейса, для которого доступна реализация в исходном коде. «Основной компонент» в этом контексте
означает важный компонент (ядро, оконная система и т. п.) конкретной операционной системы (если есть),
на которой работает исполняемое произведение, или компилятор, использованный для создания произведения,
или интерпретатор объектного кода, используемый для его запуска.

  «Соответствующий исходный код» для произведения в форме объектного кода означает весь исходный код,
необходимый для генерации, установки и (для исполняемого произведения) запуска объектного кода и модификации произведения,
включая скрипты для управления этими действиями. Однако он не включает системные библиотеки произведения,
или общие инструменты или общедоступные свободные программы, которые используются без изменений
при выполнении этих действий, но не являются частью произведения. Например, Соответствующий исходный код
включает файлы определения интерфейсов, связанные с исходными файлами произведения,
а также исходный код общих библиотек и динамически связанных подпрограмм,
которые произведение специально требует, например, из-за тесной передачи данных или управления между этими подпрограммами
и другими частями произведения.

  Соответствующий исходный код не должен включать ничего, что пользователи могут автоматически
восстановить из других частей Соответствующего исходного кода.

  Соответствующий исходный код для произведения в исходном коде — это то же самое произведение.
