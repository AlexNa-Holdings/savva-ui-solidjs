# Инициализация приложения и подключение к бэкенду

## Что такое бэкенд SAVVA (SAVVA node)?
Бэкенд SAVVA — это серверный компонент, который **индексирует/кэширует данные, полученные из активности блокчейна**, и предоставляет быстрые, удобные для UI API и методы WebSocket. Один бэкенд может обслуживать **несколько доменов SAVVA** — думайте о «домене» как об отдельной социальной сети SAVVA (брендинг, вкладки, ассеты, настройки по умолчанию), все это поддерживается одним узлом.

## Что нужно приложению при старте
При запуске веб‑приложению нужны два входных параметра:

1. **Backend URL** — базовый URL бэкенда SAVVA.
2. **Domain name** — какой домен SAVVA (социальная сеть) рендерить по умолчанию.

Значения по умолчанию берутся из маленького JSON‑файла в корне веба (YAML также поддерживается как запасной вариант):

### `/default_connect.json`
```json
{
  "domain": "savva.app",
  "backendLink": "https://ui.savva.app/api/",
  "gear": true,
  "default_ipfs_link": "ipfs://bafy.../something.json"
}
```

* `backendLink` — базовый HTTP‑эндпоинт бэкенда SAVVA (приложение нормализует его).
* `domain` — начальный домен для рендера; его можно переключить позже в UI.
* `gear` — включает элементы разработчика в UI (опционально).
* `default_ipfs_link` — опциональный удобный дефолт, используемый в некоторых потоках.

> **Примечание по формату**
> Приложение сначала пытается получить `/default_connect.json`. Если этот запрос терпит неудачу, оно откатывается к `/default_connect.yaml` для обратной совместимости. Новые деплои должны использовать JSON.

> **Примечание для продакшна**
> В продакшне этот файл обычно отдаёт ваш HTTP‑сервер (например, Nginx) и фактически **определяет, какой домен** будет показывать развернутое веб‑приложение по умолчанию. Обычный шаблон — обслуживать конкретный файл с диска:
>
> ```nginx
> # example: serve a static default_connect.json
> location = /default_connect.json {
>   default_type application/json;
>   alias /etc/savva/default_connect.json;
> }
> ```
>
> Подстройте под вашу инфраструктуру; главное, чтобы приложение могло сделать `GET /default_connect.json`.

## Последовательность загрузки

1. **Загрузка конфигурации сайта (`/default_connect.json` или `.yaml`)**  
   Приложение сначала пытается получить `/default_connect.json`; если он недоступен, откатывается к `/default_connect.yaml`. Оно валидирует `backendLink`, сохраняет `domain` и немедленно **настраивает эндпоинты** (HTTP база + WS URL) с использованием этих значений. &#x20;

2. **Настройка эндпоинтов**

   * `httpBase` — нормализованная версия `backendLink` (гарантированно завершается слэшем).
   * WS‑URL выводится из той же базы, указывая на `.../ws` (протокол меняется на `ws:` или `wss:`) и включает `?domain=...` в строке запроса.  
     Это сохраняет **единый источник правды** для HTTP и WS.&#x20;

3. **Запрос `/info`**  
   С установленными эндпоинтами приложение делает `GET <httpBase>info` и сохраняет JSON. С этого момента **/info управляет поведением во время выполнения** (домены, цепочка, IPFS, ассеты).&#x20;

4. **Вывод состояния рантайма из `/info`**  
   Используются следующие поля (см. пример ниже):

   * **`domains`** → список доступных доменов. UI предпочитает явно указанный `domain` из YAML/переопределения; если его нет в `/info`, всё равно используется конфигированный домен.&#x20;
   * **`blockchain_id`** → целевой EVM chain ID. Помощник кошелька может переключить/добавить эту сеть.&#x20;
   * **`ipfs_gateways`** → удалённые IPFS‑шлюзы, которые пробуют в порядке очереди (если не включён локальный IPFS‑override).&#x20;
   * **`assets_url`** и **`temp_assets_url`** → **база ассетов** (прод vs тест). Приложение вычисляет **префикс ассетов для активного домена** как
     `(<assets base> + <domain> + "/")` с **фоллбеком** на `/domain_default/`, если удалённый `config.yaml` отсутствует. &#x20;

5. **Загрузка ассетов и конфига домена**  
   Приложение пробует `(<active prefix>/config.yaml)` с коротким таймаутом; при ошибке откатывается к дефолтному паку `/domain_default/config.yaml`. Полученный распарсенный конфиг (логотипы, вкладки, локали и т. п.) сохраняется, и UI рендерится соответственно.&#x20;

6. **WS в рантайме**  
   WS‑клиент использует вычисленный `ws`‑URL из эндпоинтов; когда бэкенд/домен меняются, эндпоинты пересчитываются, и слой WS подхватывает изменения.&#x20;

## Пример `/info` (иллюстрация)

```json
{
  "domains": [
    "savva.app",
    {"name": "art.savva"},
    "dev.savva"
  ],
  "blockchain_id": 369,
  "ipfs_gateways": [
    "https://cloudflare-ipfs.com/ipfs/",
    "https://ipfs.io/ipfs/"
  ],
  "assets_url": "https://cdn.savva.network/assets/",
  "temp_assets_url": "https://cdn.savva.network/assets-test/"
}
```

### Поле за полем (что приложение с этим делает)

* **domains** — список выбираемых доменов. Диалог **Switch backend / domain** заполняется из `/info`, но сконфигурированный домен всё ещё имеет приоритет, если `/info` отстаёт. &#x20;
* **blockchain\_id** — числовой EVM chain ID; используется для формирования метаданных `switch/add chain` и для гарантии, что кошелёк находится в **требуемой сети**. &#x20;
* **ipfs\_gateways** — упорядоченный список удалённых шлюзов; комбинируется с опциональным перезаписью **Local IPFS** (когда включено в настройках) для формирования **активного** порядка шлюзов.&#x20;
* **assets\_url / temp\_assets\_url** — приложение поддерживает **среду ассетов** (`prod`/`test`) и выбирает соответствующую базу. Затем оно вычисляет `/<base>/<domain>/` и загружает `config.yaml`. Если удалённый пак отсутствует или медленный, используется **дефолтный** `/domain_default/`.&#x20;

## Где это находится в коде (для быстрой ссылки)

* Загрузка при старте и конфигурации сайта (`/default_connect.json` с откатом на `.yaml`), затем `/info`: **`src/context/AppContext.jsx`** и **`src/hooks/useConnect.js`**. Общий загрузчик находится в **`src/utils/loadSiteConfig.js`**. &#x20;
* Источник правды для эндпоинтов (HTTP база + WS URL): **`src/net/endpoints.js`**.&#x20;
* Разрешение списка доменов, chain ID, IPFS‑шлюзы, среда ассетов и загрузка ассетов домена: **`src/context/AppContext.jsx`**.  &#x20;
* Диалог переключения, который запрашивает `/info` и нормализует `domains`: **`src/x/SwitchConnectModal.jsx`**.&#x20;
