# Руководство по установке — Обзор

Это руководство проведёт вас через настройку собственной сети SAVVA с нуля. Следуя этим инструкциям, вы сможете запустить полноценный экземпляр SAVVA с серверной частью и пользовательским интерфейсом.

## Что вы развернёте

Полная сеть SAVVA состоит из:

1. **Сервер (Backend)** - API-сервер на Go, который обрабатывает:
   - Аутентификацию пользователей и сессии
   - Хранение и получение постов
   - Интеграцию с IPFS
   - Управление базой данных
   - WebSocket‑соединения
   - Взаимодействие с блокчейном

2. **Веб-интерфейс (UI)** - фронтенд на SolidJS, который предоставляет:
   - Пользовательский интерфейс для создания и просмотра контента
   - Интеграцию с Web3-кошельками
   - Загрузку файлов в IPFS
   - Взаимодействие со смарт‑контрактами
   - Поддержку нескольких языков
   - SEO-слой: серверно-рендеренный HTML для поисковых систем (Google, Bing, Yandex, Baidu, DuckDuckGo, Apple), AI-краулеров (GPTBot, ClaudeBot, PerplexityBot, Google-Extended, ...), и сервисов предварительного просмотра ссылок (Telegram, X, Facebook, Discord, Slack, WhatsApp, ...), с индивидуальными для каждого домена `robots.txt` и картами сайта

3. **Смарт-контракты** :
   - Контракт токена SAVVA
   - Контракт стекинга
   - Контракт управления (governance)
   - Контракт NFT для контента
   - И другие...

```
   ↓

   ↓

   ↓
5. Testing & Verification
```

## Требования — краткий обзор

- **Сервер**: сервер на Linux (рекомендуется Ubuntu 20.04+)
- **База данных**: PostgreSQL 14+ (или управляемый сервис базы данных)
- **IPFS**: локальная нода IPFS + внешняя служба пининга с публичным шлюзом
- **Веб-сервер**: Nginx или Apache
- **Домен**: доменное имя с SSL‑сертификатом
- **Блокчейн**: доступ по RPC к сети, совместимой с Ethereum (рекомендуется WSS)
