# Руководство по установке — обзор

Это руководство проведёт вас через процесс настройки собственной сети SAVVA с нуля. Следуя этим инструкциям, вы сможете запустить полноценный экземпляр SAVVA с серверной частью и пользовательским интерфейсом.

## Что вы развернёте

Полная сеть SAVVA состоит из:

1. **Сервер бэкенда** - API-сервер на Go, который обеспечивает:
   - Аутентификацию пользователей и управление сессиями
   - Хранение и получение постов
   - Интеграцию с IPFS
   - Управление базой данных
   - WebSocket-подключения
   - Взаимодействие с блокчейном

2. **Веб-интерфейс (UI)** - фронтенд на SolidJS, который предоставляет:
   - Интерфейс для создания и просмотра контента
   - Интеграцию с Web3-кошельками
   - Загрузку файлов в IPFS
   - Взаимодействие со смарт‑контрактами
   - Поддержку нескольких языков

3. **Смарт‑контракты** (опционально) - если вы запускаете новую сеть:
   - Контракт токена SAVVA
   - Контракт стейкинга
   - Контракт управления (governance)
   - Контракт NFT для контента
   - И другие...

## Порядок установки

```
1. Настройка предварительных требований
   ↓
2. Установка бэкенд-сервера
   ↓
3. Настройка веб-интерфейса (UI)
   ↓
4. Конфигурация
   ↓
5. Развертывание
   ↓
6. Тестирование и проверка
```

## Обзор требований

- **Сервер**: Linux-сервер (рекомендуется Ubuntu 20.04+)
- **База данных**: PostgreSQL 14+ (или управляемый сервис базы данных)
- **IPFS**: локальный узел IPFS + внешний сервис закрепления (pinning) с публичным шлюзом
- **Веб‑сервер**: Nginx или Apache
- **Домен**: имя домена с SSL‑сертификатом
- **Блокчейн**: доступ по RPC к сети, совместимой с Ethereum (рекомендуется WSS)
